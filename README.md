### 삼성 헬스 데이터 접근 브릿지 앱 

### 배경
- 삼성 헬스 데이터는 앱을 통해서만 접근 가능
- 삼성 헬스 데이터는 http rest api 없음
- 구글 헬스 커넥트 앱이 존재함
  - 여러 헬스 신체 정보 데이터를 이어주는 구글 공식 앱
  - 구글핏, 삼성헬스, 인바디, 워치 등 다양한 기기와 앱을 연동하는 목적
  - 장점 : 
    - 구글 공식
    - 지원하는 헬스 앱이 매우 많음
  - 단점 : 
    - 헬스 커넥트 앱을 설치하고 연동 시작한 시점 부터 연동 됨
    - 즉, 헬스 커넥트 앱 설치 이전 과거 데이터 연동 불가
- 번외
  - 삼성 헬스와 거의 유사한 구글 피트니스 (구글fit) 앱이 존재함
  - 과거 2023(?) 웹 api 제공했지만, 차단
  - 구글핏 웹 버전도 지원 중단
- 시도1
  - 구글 시트 > 확장 프로그램 > 구글핏 > 구글 커넥트 > 삼성 헬스
  - 구글 시트 확장 프로그램이 구글에 차단됨
- 시도2
  - 구글 개발자 oauth api > 구글 커넥트 > 삼성 헬스
  - 구글 개발자 oauth api 테스트를 통해서 구글핏에 데이터 CRUD 가능
  - 하지만, 구글핏 앱에서 데이터가 조회 되지 않음
  - oauth api 를 통해서만 데이터 조회됨
  - 같은 구글 계정이라도 데이터 스트림 id 가 다른지 원인 파악 불가

### 결론
- 삼성 헬스 sdk 를 사용하는 브릿지 앱을 만든다
- 데이터 흐름 : 데이터 입력 > 브릿지 앱 > 삼성 헬스
